# Arquitectura y Reglas del Proyecto (MVPs FinTech)

Última actualización: Hito de Unificación de Marca y Optimización SEO/SEM (Abril 2026)
Modo de proyecto: ESTRICTO

## 1. Componente Visivo (Toasts)
- **Regla Estricta:** PROHIBIDO depender de `alert()`, `window.confirm()` o `console.log` para feedback visual al usuario.
- **Implementación:** Se debe utilizar exclusivamente la función global `showToast(message, type)` definida en `app.js` basada en creación al vuelo de DOM y clases de Tailwind CSS nativas (`bg-green-600` o `bg-red-600` con `opacity` y `translate-y`).
- **Estados de Botón:** Todo envío a la API (formularios) obliga a que el botón se deshabilite, cambie a `opacity-50 pointer-events-none` y su texto a "Procesando..." durante el fetch.

## 2. Modo Suplantación Blindado
- **Origen de datos:** Durante la suplantación (`impersonatingId` presente en `localStorage`), la identidad de UI (Sidebar y Vista de Configuración) **NO DEBE** leer el `userStr` local, sino que debe consultar estrictamente al endpoint de perfil `/admin/get_user.php?userId=X`.
- **Modo Sólo Lectura:** Los inputs de configuración personal de un cliente suplantado deben mostrarse inmediatamente como `readonly` y con estilizado opaco (`bg-gray-50 text-gray-500`) para blindar contra ediciones por error del administrador superior. Y el botón de guardado debe estar bloqueado y oculto.

## 3. APIs y Nomenclatura Estricta
- **Nomenclatura (JSON payload):** Todo el backend y frontend respeta estrictamente CamelCase para las variables compuestas. Ejemplos obligatorios: `userId`, `riskProfile`, `annualReturnRate`, `investmentPeriod`, `nextInvestmentDate`.
- **Nuevos Endpoints (Segregación de Lógica):** La actualización de variables operativas de inversión (perfil financiero) debe dispararse exclusivamente hacia `api/admin/update_financial_profile.php` por el método HTTP `PUT`. No mezclar con el endpoint de settings tradicionales.

## 4. Prevención de Rage Clicks y Burbujas en Tablas
- **Blindaje SVG:** Todos los elementos `<svg>` o `<path>` inyectados dentro de tags `<button>` en las filas dinámicas de una tabla DEBEN poseer la clase de Tailwind `pointer-events-none`.
- **Event Binding:** Para prevenir colisiones de onclick (Bubbling) entre un `<tr>` principal y un `boton` de acción (como el engrane, ojo o eliminar), los botones deben tener sus eventos mapeados en línea: `onclick="funcionDeAccion(event, args)"`.
- **Regla de oro de Eventos:** Su primera línea adentro siempre debe ser matemáticamente `event.stopPropagation();` para blindar el trigger.

## 7. Arquitectura de Growth Hacking "Agencia 360" (Doble MCP)
- **Visión General:** El proyecto ha evolucionado para incorporar un "Cerebro Orquestador" que conectará directamente Antigravity con Google Drive y Meta Ads para gestionar campañas de forma autónoma.
- **Data Mirroring (Espejo de Datos):** 
  - La lógica financiera y los costos ocultos residen en la hoja `INTERNO`.
  - El cliente solo tiene acceso a `DASHBOARD` a través del script de Google Apps Script que empuja los datos planos y unificados.
- **Estado de Infraestructura (Abril 2026):**
  - **Google Drive MCP:** ✅ [CONFIGURADO] Antigravity cuenta con acceso OAuth completo mediante `credentials.json` y `token.json` locales. El servidor MCP fue inyectado en `mcp_config.json` exitosamente.
  - **Meta Ads MCP:** ⏳ [PENDIENTE] Programado para la próxima sesión. Implicará extraer métricas (Gasto, Leads, ROAS) y cruzarlas matemáticamente con el MCP de Google Sheets.
- **Seguridad/Cookies:** La clave de autenticación (`fintech_token`) se mantuvo para garantizar continuidad en la base de datos de usuarios heredados sin invalidar sesiones.
- **Componentes Nativos:** La UI de la landing utiliza `<details>` de HTML5 para los FAQs y Vanilla JS (con `scrollBy` y `setInterval`) para el auto-slide del carrusel, logrando cero deuda técnica.
- **Seguimiento SEM Cero-GTM:** Se inyectaron `gtag` y `clarity` en la cabecera. Todos los botones CTA disparan nativamente un evento de GA4 (`generate_lead`) evitando depender de Google Tag Manager para la optimización de pautas.

## 7. Herramienta de Growth Hacking (Ecosistema "Agencia 360" & MCP Dual)
- **Visión Operativa (Espejo de Datos):** El sistema actual se basa en Google Drive/Sheets aislando los márgenes de la agencia. Existe un `INTERNO_[Cliente]` (motor contable privado), un `DASHBOARD_[Cliente]` (vista de cliente compartida sin fórmulas) y un `ADMIN_MAESTRO_AGENCIA` (torre de control).
- **El Rol de Antigravity (Fase 2 y 3):** En lugar de sobrecargar Google Apps Script (`UrlFetchApp`) con lógica compleja de APIs, Antigravity actuará como el "Cerebro Orquestador" mediante un modelo de "Doble MCP" (Model Context Protocol):
  1. **Meta Ads MCP:** Extracción en tiempo real de métricas (Gasto, Leads, ROAS).
  2. **Google Sheets/Drive MCP:** Antigravity leerá el presupuesto desde el archivo `INTERNO`, cruzará los datos con Meta Ads, calculará el CPL real (ocultando el fee de la agencia), y escribirá los valores finales. Luego, el script actual de Sheets empujará los datos al `DASHBOARD`.
- **Ventajas Proactivas:** Este ecosistema permite tener "Agentes de Monitoreo" que escanean cada 4 horas buscando anomalías (ej. picos de CPL), lanzan alertas, y generan resúmenes ejecutivos con IA (Gemini 3.1 Pro) redactados directamente en la hoja de reporte del cliente.
